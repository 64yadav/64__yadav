Workflow
Data Analytics Ka Sahi Process

"Data analyst ka 80% time data ko samajhne mein jaata hai, sirf 20% actual analysis mein. Yahi reality hai."

— Industry standard breakdown
1
📌 Problem Define Karo (Sabse Pehle)
⏱ 10–15 min invest karo
Analysis shuru karne se pehle ek line mein likho: "Main ye janana chahta hoon ki [X], taaki [Y] decision le sakun." Bina clear question ke analysis waste hai.
Pro TipStakeholder se poochho: "Agar ye analysis perfect result de, to aap kya differently karoge?" — Yahi tumhara actual goal hai.
2
📥 Data Collect & Understand Karo
⏱ 20–30% total time
Data kahan se aa raha hai? Kitna reliable hai? Koi missing values hain? Duplicate rows? Date format consistent hai?
df.info() df.describe() df.isnull().sum() df.duplicated().sum() df.dtypes
Avoid ThisData ko bina dekhe seedha analysis mat shuru karo. Garbage in = Garbage out. 1 ganda row analysis barbad kar sakta hai.
3
🧹 Data Clean Karo
⏱ 30–40% total time (sabse time-consuming)
Missing values handle karo, wrong data types fix karo, outliers deal karo, inconsistent categories fix karo (e.g. "Mumbai" vs "mumbai" vs "MUMBAI").
Missing ValuesNumeric → Median fill karo (outliers safe). Categorical → Mode ya "Unknown". 50%+ missing → Column drop karo.
OutliersRemove mat karo blindly. Pehle samjho — data entry error hai ya real extreme value? Business context maayene rakhta hai.
df.fillna() df.dropna() str.lower().strip() pd.to_datetime()
4
🔍 EDA — Explore Karo (Visualize First!)
⏱ 20–25% total time
Patterns dhundho, distributions dekho, relationships explore karo. Har analysis visualize se shuru karo — numbers akele liars hote hain.
EDA Sequence — Is Order Mein Karo
StepKya KaroTool
UnivariateHar variable ka distribution dekhohistogram, boxplot, countplot
Bivariate2 variables ka relationshipscatter, heatmap, bar comparison
MultivariateMultiple variables togetherpairplot, correlation matrix
Time-basedTrend dekho (agar date hai)line chart, rolling avg
Secret WeaponPandas Profiling ya ydata-profiling se ek line mein complete EDA report bana lo: ProfileReport(df).to_file("eda.html") — 10 minute ka kaam 2 min mein!
5
📊 Analysis Karo
⏱ 15–20% total time
Ab actual statistical analysis. Problem ke hisaab se method choose karo. Descriptive → Diagnostic → Predictive → Prescriptive — isi order mein sochho.
4 Types of Analytics — Seedhi Baat
TypeQuestionExample
DescriptiveKya hua?Last month sales kitne the?
DiagnosticKyun hua?Sales kyun giri — price, ads, ya competitor?
PredictiveKya hoga?Agla month sales kitna hoga?
PrescriptiveKya karna chahiye?Kaunsa action lena chahiye?
6
📢 Communicate Karo — Story Bolo, Data Nahi
⏱ 10–15% time — lekin 80% impact!
Sab se important aur sab se neglected step. Jo analyst clearly communicate kare, wahi successful hota hai — chahe analysis average ho.
Golden RuleExecutive ko: 1 slide = 1 insight. Pehle conclusion bolo, phir data. "Revenue 23% gira, main reason cart abandonment hai" — phir charts.
AvoidData dump mat karo. 15 charts ki PDF mat bhejo. Audience ne poochha nahi tha, to full technical details mat do.

🔄 Templates Banao, Baar Baar Mat Likho

EDA, cleaning, reporting ke liye templates banao. Har project mein copy-paste karo aur customize karo.

def quick_eda(df):
print(df.info())

📋 SQL Mein Pehle Filter Karo

Saara data Python mein load mat karo. SQL mein WHERE, GROUP BY karo — sirf zaroori data lo. Network aur RAM dono bachega.

SELECT city, SUM(sales)
FROM orders
WHERE year = 2024
GROUP BY city
10x faster

🤖 Profiling Tools Use Karo

Manual EDA mat karo. ydata-profiling ya sweetviz ek line mein complete report bana deta hai.

from ydata_profiling
import ProfileReport
ProfileReport(df)
3x faster

⌨️ Keyboard Shortcuts Yaad Karo

Excel: Ctrl+Shift+L (filter), Ctrl+T (table). Jupyter: A/B (cell add), DD (delete), Shift+Enter (run). Ek hafte practice → lifetime time save.

04
Differentiation
Bheed Se Alag Dikhne Ke Real Tarike

"Average analyst data dikhata hai. Achha analyst insights dikhata hai. Great analyst decisions drive karta hai."

🎯 Insight, Not Just Data

Sirf numbers mat do. "Sales 20% giri" nahi — "Sales 20% giri kyunki Product X return rate 35% hai — ye fix karo." Business impact wala conclusion do.

📖 Narrative Banao

Har presentation ek kahani honi chahiye: Problem → Evidence → Insight → Recommendation. Charts mein bhi title sirf label nahi, insight likhao: "Mumbai Outperforms by 40%" — not just "City Comparison".

🤔 Assumptions Batao

Jo analyst openly bataye "maine ye assume kiya" — woh trusted hota hai. Assumptions chhupa ke confidence dikhana backfire karta hai baad mein.

❓ Limitations Acknowledge Karo

"Ye analysis sirf last 6 months ka data hai, seasonal effect ho sakta hai" — ye bolna weakness nahi, professionalism hai. Log trust karte hain.

🎨 Visualization Ka Taste Badhao

Default Excel charts mat use karo. Clean, minimal charts jyada powerful hote hain. Ek achha chart ek kharab presentation ko bhi rescue kar sakta hai.

💬 Proactively Communicate Karo

"Analysis mein 2 more days lagenge" — ye wait karne ki jagah pehle batao. Stakeholder management = analyst ki superpower. Technical skills se zyada career impact.

🔁 Reproducible Work Karo

Tera koi bhi code ya analysis koi bhi ek button click se dobara chala sake. Documentation + clean code = professional identity.

📊 Right Chart Choose Karo

Comparison → Bar. Trend → Line. Part of whole → Pie (max 5 slices). Distribution → Histogram. Relationship → Scatter. Galat chart = galat message.

05
Warning Zone
Ye Galtiyan Bilkul Mat Karo

❌ Data Ko Bina Samjhe Use Karna

Fix: df.info() + df.describe() pehle

Directly analysis karna without understanding data types, missing values, date ranges. Ye sabse common aur sabse badi galti hai. Ek wrong column type pura calculation galat kar deta hai.

❌ Correlation = Causation Samajhna

Fix: "Associated with" bolna

"Ice cream sales badhne se crime badhta hai" — dono summer mein badhte hain. Sirf isliye ki dono trends match karte hain, iska matlab ek doosre ko cause nahi karta. Ye presentation mein bolna career risk hai.

❌ Misleading Visuals Banana

Fix: Y-axis zero se start karo

Y-axis ko 95 pe start karna taaki chhoti si change dramatic lage. Truncated axes, 3D pie charts, dual-axis charts without proper labeling — ye trust todta hai jab koi notice kare.

❌ Small Sample Size Pe Big Claims

Fix: Sample size hamesha mention karo

50 responses pe "Customer 80% satisfied hai" bolna aur ye 5 lakh customers pe apply karna. Statistical significance check karo. Small n = Low confidence.

❌ Raw Numbers Dena Without Context

Fix: Benchmark, previous period, target ke saath compare karo

"10,000 users aa gaye" — ye accha hai ya bura? Last month kya tha? Target kya tha? Industry average kya hai? Context ke bina number meaningless hai.

❌ Outliers Ko Blindly Remove Karna

Fix: Outlier ka reason dhundho pehle

Data entry error alag cheez hai aur genuine extreme value alag. Ek fraud transaction outlier hai — use remove kiya to fraud miss ho jayega. Business context zaroori hai.

❌ Overfitting Model Banana

Fix: Train-Test split + Cross-validation

Training data pe 99% accuracy aur real data pe 60% — model ne data yaad kar liya, seekha nahi. Hamesha unseen test data pe validate karo model ko.

❌ Stakeholder Ka Context Na Lena

Fix: 15 min discovery call pehle

3 din analysis karke woh result dena jo stakeholder chahta hi nahi tha. "Mujhe region wise breakdown chahiye tha, city nahi." — Clarify first, code later.

❌ Work Document Nahi Karna

Fix: Minimum README ya comments likho

6 mahine baad apna hi code samajh nahi aata. Ya team member ne poocha "ye column kahan se aaya?" aur tum bhi nahi jaante. Documentation = future self ka gift.

Before You Submit
Har Project Ka Final Checklist
✅ Analysis Shuru Karne Se Pehle
  • ☐Business question clearly define hai?
  • ☐Data source reliable hai?
  • ☐Data ka time period define hai?
  • ☐Stakeholder ki expectations clear hain?
  • ☐Deliverable format decide hai? (dashboard/report/PPT)
  • ☐Deadline confirm hai?
✅ Data Cleaning Ke Baad
  • ☐Missing values handle kiye?
  • ☐Duplicates remove kiye?
  • ☐Data types correct hain?
  • ☐Outliers checked aur documented?
  • ☐Original raw data preserved hai?
  • ☐Row count before/after note kiya?
✅ Analysis Ke Baad
  • ☐Results sense make karte hain (sanity check)?
  • ☐Key findings 3 points mein summarize kar sakte ho?
  • ☐Assumptions documented hain?
  • ☐Limitations mention kiye?
  • ☐Charts misleading toh nahi?
  • ☐Numbers contextualized hain (vs target/last period)?
✅ Submit Karne Se Pehle
  • ☐Non-technical audience ko bhi samajh aayega?
  • ☐Recommendation clear hai (not just findings)?
  • ☐Spelling/grammar check kiya?
  • ☐Data source mentioned hai?
  • ☐File names clean hain?
  • ☐Kisi colleague ko ek baar dikhaya?
Summary: Karo vs Mat Karo

✓ Ye Karo — Hamesha

  • Pehle question define karo, phir data dekho
  • Raw data kabhi overwrite mat karo
  • Insight ke saath recommendation do
  • Simple charts > Complex charts
  • Business language bolao, technical jargon nahi
  • Assumptions openly batao
  • Results cross-verify karo common sense se
  • Template banao baar baar kaam ke liye

✗ Ye Mat Karo — Kabhi Nahi

  • Data dump karo without story
  • Correlation ko causation bolo
  • Y-axis truncate karo impression ke liye
  • Missing values explain nahi karo
  • Small sample pe big generalization karo
  • Analysis bina test kiye live karo
  • Stakeholder se halfway mein poocho "kya chahiye"
  • Pie chart mein 7+ slices rakho
