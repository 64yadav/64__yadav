Data Clean Karna — Power Query

90% analysis ki quality yahaan decide hoti hai
Power Query mein Kab Kya Karo
🔍 Step 1 — Check karo Column Quality, Column Distribution, Column Profile — View tab mein enable karo
🗑️ Step 2 — Hatao Extra columns, duplicate rows, empty rows
✏️ Step 3 — Theek karo Data types, NULL values, text case, spaces
⚙️ Step 4 — Transform karo Columns split karo, merge karo, pivot/unpivot karo
ProblemPower Query mein Kaise Theek KareinKab Zarurat Padti Hai
Wrong data typeColumn select → Data Type change karoNumbers text mein aa jaate hain
NULL / blank valuesReplace Values → null ko 0 ya "Unknown" karoCalculation mein error aata hai
Extra spacesTransform → TrimCity "Delhi " aur "Delhi" alag dikhte hain
Text case problemTransform → Capitalize / Uppercase / Lowercase"mumbai" aur "Mumbai" alag group banta hai
Duplicate rowsHome → Remove DuplicatesSame order 2 baar aa raha hai
Column split karnaTransform → Split Column → By Delimiter"First Last" name ko alag karna ho
Date format galatData type Date set karo, locale change karoDate sort sahi nahi hoti
Unpivot karnaColumns select → Transform → UnpivotMonths column mein hain, rows mein chahiye
Har step Power Query mein "Applied Steps" mein save hoti hai. Kabhi bhi undo kar sakte ho — is liye Power Query mein karo, source file mein haath mat lagao.
✅ Golden Rule Jab bhi data aaye — pehle Power Query mein jaao, column profile dekho, tab "Close & Apply" karo. Seedhe report view pe mat jaao.

TypeMatlabExample

DAX Formulas — Complete Guide

Data Analysis Expressions — Power BI ki asli taakat
SUM / COUNT / AVERAGE / MIN / MAX
Basic aggregation — sabse pehle yeh seekho
DAX
-- Total Revenue
Total Revenue = SUM(Sales[Amount])

-- Total Orders count
Total Orders = COUNT(Sales[Order_ID])

-- Unique customers count
Unique Customers = DISTINCTCOUNT(Sales[Customer_ID])

-- Average order value
Avg Order Value = AVERAGE(Sales[Amount])
✅ Kab Use KaroKPI cards, summary numbers, total revenue, total units sold dikhane ke liye.
CALCULATE
DAX ka sabse powerful function — filter lagake calculate karo. Yeh aana hi chahiye.
DAX
-- Sirf Delhi ki sales
Delhi Sales = 
  CALCULATE(
    SUM(Sales[Amount]),
    Sales[City] = "Delhi"
  )

-- 2024 ki total sales
Sales 2024 = 
  CALCULATE(
    SUM(Sales[Amount]),
    YEAR(Sales[Date]) = 2024
  )

-- All filters hata ke (grand total)
Total Sales All = 
  CALCULATE(
    SUM(Sales[Amount]),
    ALL(Sales)
  )
✅ Kab Use KaroJab filter ke saath calculate karna ho — specific category, region, time period ki sales. % contribution bhi CALCULATE se milti hai.
IF / SWITCH
Condition ke hisaab se alag value — Excel ke IF jaisa
DAX
-- Sales category
Sales Category = 
  IF(Sales[Amount] > 50000, "High", "Low")

-- Multiple conditions ke liye SWITCH better hai
Performance = 
  SWITCH(
    TRUE(),
    Sales[Amount] >= 100000, "🟢 Excellent",
    Sales[Amount] >= 50000,  "🟡 Good",
    Sales[Amount] >= 20000,  "🟠 Average",
    "🔴 Low"
  )
✅ Kab Use KaroColor coding ke liye, performance buckets banana ho (High/Medium/Low), conditional labels dikhane ho.
Time Intelligence Functions ⭐
Date Table hona zaroori hai inke liye — yeh functions interviews mein pooche jaate hain
DAX
-- Year-to-date sales (jan se aaj tak)
Sales YTD = 
  TOTALYTD(SUM(Sales[Amount]), Dates[Date])

-- Pichle mahine ki sales
Sales Last Month = 
  CALCULATE(
    SUM(Sales[Amount]),
    PREVIOUSMONTH(Dates[Date])
  )

-- Month-over-Month Growth %
MoM Growth % = 
  VAR CurrentMonth = SUM(Sales[Amount])
  VAR LastMonth = 
    CALCULATE(
      SUM(Sales[Amount]),
      PREVIOUSMONTH(Dates[Date])
    )
  RETURN
    DIVIDE(CurrentMonth - LastMonth, LastMonth, 0)

-- Same period last year
Sales SPLY = 
  CALCULATE(
    SUM(Sales[Amount]),
    SAMEPERIODLASTYEAR(Dates[Date])
  )
✅ Kab Use KaroYTD, MTD, Year-over-Year comparison, growth trend — business mein yeh hamesha poochhte hain. Date Table zaroori hai — bina date table ke kaam nahi karega.
⚠️ Date Table Banao PehleMark as Date Table karo Model View mein. TOTALYTD aur baaki time functions tabhi kaam karte hain.
DIVIDE / % Contribution
Division hamesha DIVIDE se karo — zero se divide hone ka error avoid hota hai
DAX
-- Safe division
Profit Margin % = 
  DIVIDE(Sales[Profit], Sales[Revenue], 0)

-- % contribution (is category ka total ka kitna %)
Sales % of Total = 
  DIVIDE(
    SUM(Sales[Amount]),
    CALCULATE(SUM(Sales[Amount]), ALL(Sales)),
    0
  )
✅ Kab Use KaroProfit margin, conversion rate, % contribution — koi bhi ratio ya percentage calculate karna ho.
RANKX / TOPN
Ranking nikalna — Top performers dhundhna
DAX
-- Product rank by sales
Product Rank = 
  RANKX(
    ALL(Products[Product_Name]),
    SUM(Sales[Amount]),
    ,
    DESC
  )

-- Filter: sirf top 5 dikhao
Top 5 Filter = 
  IF([Product Rank] <= 5, 1, 0)
✅ Kab Use Karo"Top 10 products", "Best performing salespeople" — leaderboard type visuals ke liye.
VAR ... RETURN
Complex formulas ko readable aur fast banata hai — advanced level
DAX
-- Variables use karke readable formula
YoY Growth % = 
  VAR SalesCY = SUM(Sales[Amount])
  VAR SalesPY = 
    CALCULATE(
      SUM(Sales[Amount]),
      SAMEPERIODLASTYEAR(Dates[Date])
    )
  RETURN
    DIVIDE(SalesCY - SalesPY, SalesPY, 0)
✅ Kab Use KaroJab ek formula mein same calculation baar baar aaye — VAR mein store karo, ek baar calculate hoga, fast bhi aur readable bhi.

Measures vs Calculated Columns

Yeh confusion sabko hota hai — ek baar samajh lo
📐 Measure Recommended

Context ke hisaab se dynamically calculate hota hai. File size nahi badhata.

DAX
Total Sales = 
  SUM(Sales[Amount])
✅ Use Karo JabCharts, cards, tables mein aggregated value chahiye — SUM, COUNT, AVERAGE, CALCULATE.
📋 Calculated Column Soch ke Use Karo

Har row ke liye ek value calculate karke table mein add hoti hai. File size badhta hai.

DAX
Full Name = 
  Customers[First] 
  & " " 
  & Customers[Last]
✅ Use Karo JabRow-level calculation — jaise Full Name banana, category assign karna, ya filter column banana.
99% kaam Measures se hota hai. Calculated Column sirf tab banao jab slice/filter karna ho us value pe.

Charts & Visualizations — Kab Kaun Sa Use Karo

Galat chart = galat insight. Sahi chart choose karna skill hai.
📊 Bar Chart / Column Chart Sabse Common
Kab use karo: Categories ki comparison karna ho — kaunsa product zyada bika, kaunsi city top hai.
✅ Example: "City-wise Total Revenue", "Product Category Sales Comparison"
❌ Avoid: Time trend dikhane ke liye mat use karo
📈 Line Chart Trend ke liye
Kab use karo: Time ke saath trend dikhana ho — monthly sales, weekly growth.
✅ Example: "Monthly Revenue Trend", "Daily Active Users over Time"
❌ Avoid: Categories compare karne ke liye mat use karo
🥧 Pie / Donut Chart Sirf % ke liye
Kab use karo: Sirf jab parts-of-whole dikhana ho — aur categories 5 se kam hon.
✅ Example: "Revenue by Region (4 regions)", "Product Mix %"
❌ Avoid: 5+ categories mein, ya exact comparison chahiye ho tab
🗺️ Map Visual Geography ke liye
Kab use karo: City, State, Country ke hisaab se data dikhana ho.
✅ Example: "State-wise Sales India Map", "City-wise Customer Distribution"
❌ Avoid: Non-geographic data pe — bakwaas lagta hai
🎯 KPI Card / Card Visual Har Dashboard mein
Kab use karo: Single important number dikhana — Total Revenue, Total Orders, Profit %.
✅ Example: "₹12.5L Total Revenue", "1,234 Orders This Month"
❌ Avoid: Detailed breakdown ke liye
🌲 Treemap Hierarchy ke liye
Kab use karo: Hierarchy mein proportions dikhane ho — Category → Sub-category.
✅ Example: "Product Category → Sub-category Revenue Share"
❌ Avoid: Trend ya time-based data ke liye
🌊 Waterfall Chart Finance ke liye
Kab use karo: Kahan se kahan aaye — additions aur subtractions dikhana.
✅ Example: "Net Profit Breakdown", "Budget vs Actual Variance"
❌ Avoid: Simple comparison ke liye — bar chart better hai
🔵 Scatter Chart Correlation ke liye
Kab use karo: Do variables ka relationship dikhana — kya price badhne se sales girti hai?
✅ Example: "Marketing Spend vs Revenue", "Price vs Units Sold"
❌ Avoid: Categories compare karne ke liye
📋 Table / Matrix Detail ke liye
Kab use karo: Exact numbers chahiye ho — jab chart se detail nahi milti. Matrix pivot table jaisi hoti hai.
✅ Example: "Product-wise Monthly Sales Matrix", "Detailed Order List"
❌ Avoid: Dashboard ka main visual mat banao — supporting detail ke liye rakho
Ek dashboard mein maximum 5-6 visuals rakho. Zyada visuals = confusion. Ek page, ek story.

Filters & Slicers

Dashboard ko interactive banana
ToolKya karta haiKab use karo
SlicerClick karke filter karo — buttons ya dropdownDate, City, Category filter — user interaction
Filter PaneVisual, Page, ya Report level filterDeveloper ne set kiya hua — user nahi dekhta
Date SlicerDate range select karoHamesha date filter hona chahiye dashboard mein
Drill-throughEk visual pe click karke detail page pe jaoSummary se detail pe jaana — advanced dashboards
Cross-filterEk visual click karne se doosre visuals filter ho jaate hainDefault Power BI mein hota hai — on/off kar sakte ho
✅ Best Practice Har dashboard mein Date Slicer aur main Category Slicer zaroor rakho. User ko khud filter karne do — wo khud apna answer dhundhega.

Dashboard Design Best Practices

Professional dashboard aur amateur mein yahi fark hota hai
✅ Karo

📌 Top pe KPI cards rakho
📌 Consistent colors use karo (2-3 max)
📌 Date slicer hamesha rakho
📌 Mobile view bhi check karo
📌 Title aur subtitles likho
📌 Numbers format karo (₹1.2L not 120000)
📌 White space do — crowded mat karo

❌ Mat Karo

❌ 3D charts — misleading hote hain
❌ 7+ colors ek chart mein
❌ Pie chart with 6+ categories
❌ Bina title ke visuals
❌ Ek page mein 10+ visuals
❌ Default blue color har jagah
❌ Unlabeled axes

🎨 Color Strategy
UseColorKyun
Primary metric / highlightEk strong color — blue ya orangeEye ko direct karo
Positive / growthGreenUniversal samajh
Negative / declineRedUniversal samajh
