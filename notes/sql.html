01

SQL Kya Hai?

Structured Query Language — database se baat karne ki language

SQL ek language hai jo database mein stored data ko dhundhne, filter karne, group karne aur analyze karne ke kaam aati hai. Data Analyst ka 70% kaam SQL se hota hai.

✅ Kab Use Karo Jab bhi kisi table ya database mein data dekhna, nikalna ya analyze karna ho — Excel se bada dataset ho tab especially.

🏢 Real Life Example

Ek company ki sales table hai — 1 lakh rows. Excel crash kar jaega. SQL mein 2 second mein answer aata hai.

📊 Data Analyst mein

Reports banana, dashboards ke liye data nikalna, business questions ka jawab dena — sab SQL se.

02

SELECT — Data Dekhna

Sabse basic aur sabse zaroori command

Syntax

SQL 
-- Saari columns dekhna
SELECT * FROM sales;

-- Specific columns dekhna
SELECT name, city, amount 
FROM sales;

-- Column ka naam badalna (alias)
SELECT name AS "Customer Name", 
       amount AS "Sale Amount"
FROM sales;

-- Duplicate rows hatana
SELECT DISTINCT city FROM sales;
✅ Kab Use Karo Table ka data dekhna ho, specific columns chahiye ho, ya duplicate values check karni ho.
DISTINCT use karo jab unique values dekhni ho — jaise "kitne alag-alag cities hain?"
03

WHERE — Filter Karna

Sirf wahi rows jo condition match karein
SQL 
-- Ek condition
SELECT * FROM sales
WHERE city = 'Mumbai';

-- Number filter
SELECT * FROM sales
WHERE amount > 50000;

-- Multiple conditions
SELECT * FROM sales
WHERE city = 'Delhi' AND amount > 10000;

-- Ya to yeh, ya woh
SELECT * FROM sales
WHERE city = 'Delhi' OR city = 'Mumbai';

-- List mein se dhundhna
SELECT * FROM sales
WHERE city IN ('Delhi', 'Mumbai', 'Pune');

-- Range mein dhundhna
SELECT * FROM sales
WHERE amount BETWEEN 5000 AND 20000;

-- Text search (LIKE)
SELECT * FROM customers
WHERE name LIKE 'Raj%';  -- Raj se shuru

-- NULL values dhundhna
SELECT * FROM sales
WHERE email IS NULL;

04

ORDER BY — Sort Karna

Data ko sequence mein arrange karo
SQL 
-- Ascending (chhote se bade) — default
SELECT name, amount FROM sales
ORDER BY amount ASC;

-- Descending (bade se chhote)
SELECT name, amount FROM sales
ORDER BY amount DESC;

-- Top 5 sales dekhna
SELECT name, amount FROM sales
ORDER BY amount DESC
LIMIT 5;
✅ Kab Use Karo "Top 10 products", "Sabse zyada sale karne wala", "Latest orders pehle dikhao" — in sab mein.
05

GROUP BY — Group Karke Summarize Karo

Data analyst ka sabse powerful tool

GROUP BY tab use hota hai jab "category ke hisaab se" kuch count ya sum karna ho.

SQL 
-- Har city mein kitni sales?
SELECT city, COUNT(*) AS total_orders
FROM sales
GROUP BY city;

-- Har city mein total amount
SELECT city, SUM(amount) AS total_revenue
FROM sales
GROUP BY city;

-- HAVING — grouped data pe filter
-- (WHERE group pe kaam nahi karta)
SELECT city, SUM(amount) AS total_revenue
FROM sales
GROUP BY city
HAVING SUM(amount) > 100000;

-- Month ke hisaab se sales
SELECT MONTH(sale_date) AS month,
       SUM(amount) AS monthly_revenue
FROM sales
GROUP BY MONTH(sale_date)
ORDER BY month;
⚠️ Yaad Rakho WHERE row-level filter karta hai (GROUP BY se pehle). HAVING group-level filter karta hai (GROUP BY ke baad).
✅ Kab Use Karo "City-wise revenue", "Product category mein kitni biki", "Month-wise performance" — kisi bhi "category ke hisaab se" sawaal ka jawab GROUP BY se milega.
06

JOINs — Do Tables Ko Jodna

Real world mein data hamesha multiple tables mein hota hai
JOIN TypeKya karta haiKab use karo
INNER JOINDono tables mein match karne wali rowsCommon data chahiye ho
LEFT JOINLeft table ki sab rows + right ka matchSab customers, chahe order ho ya na ho
RIGHT JOINRight table ki sab rows + left ka matchKam use hota hai
FULL JOINDono tables ki sab rowsData compare karna ho
SQL 
-- INNER JOIN: Customer naam ke saath order dekhna
SELECT c.name, o.order_id, o.amount
FROM customers c
INNER JOIN orders o 
  ON c.customer_id = o.customer_id;

-- LEFT JOIN: Sab customers, chahe order na kiya ho
SELECT c.name, o.order_id
FROM customers c
LEFT JOIN orders o 
  ON c.customer_id = o.customer_id;

-- Woh customers jo kabhi order nahi kiya
SELECT c.name
FROM customers c
LEFT JOIN orders o 
  ON c.customer_id = o.customer_id
WHERE o.order_id IS NULL;
INNER JOIN sabse zyada use hota hai. LEFT JOIN tab use karo jab left table ke sab records chahiye hon, chahe right table mein match ho ya na ho.
07

Aggregate Functions

Numbers pe calculation karne ke liye
SQL 
SELECT
  COUNT(*)           AS total_rows,      -- Total rows
  COUNT(email)      AS with_email,      -- NULL chhod ke count
  SUM(amount)       AS total_revenue,   -- Jod
  AVG(amount)       AS avg_order,       -- Average
  MAX(amount)       AS highest_sale,    -- Sabse badi value
  MIN(amount)       AS lowest_sale      -- Sabse chhoti value
FROM sales;
✅ COUNT use karo jab Kitne orders, kitne customers, kitne records — count karna ho
✅ SUM use karo jab Total revenue, total quantity — jod karna ho
08

Subquery — Query ke andar Query

Complex sawaalon ka jawab

Jab ek query ka result doosri query mein use karna ho.

SQL 
-- Average se zyada amount wali sales
SELECT * FROM sales
WHERE amount > (
  SELECT AVG(amount) FROM sales
);

-- Woh products jo kabhi bika hi nahi
SELECT product_name FROM products
WHERE product_id NOT IN (
  SELECT DISTINCT product_id FROM orders
);
✅ Kab Use Karo Jab pehle ek cheez calculate karni ho, phir us result pe kaam karna ho. "Average se zyada", "Top performers mein se" — aisi queries mein.
09

Window Functions — Advanced Level

Interview mein alag dikhna hai toh yeh seekho

GROUP BY se alag — yeh rows collapse nahi karta. Har row ke saath calculation hoti hai.

SQL 
-- Har salesperson ki rank (amount ke hisaab se)
SELECT 
  name, 
  amount,
  RANK() OVER (ORDER BY amount DESC) AS rank
FROM sales;

-- Har city mein rank
SELECT 
  name, city, amount,
  RANK() OVER (PARTITION BY city ORDER BY amount DESC) AS city_rank
FROM sales;

-- Running total (cumulative sum)
SELECT 
  sale_date, amount,
  SUM(amount) OVER (ORDER BY sale_date) AS running_total
FROM sales;
✅ Kab Use Karo Rankings, running totals, moving averages — jab GROUP BY se kaam na chale tab. Interviews mein bahut poochha jaata hai.
